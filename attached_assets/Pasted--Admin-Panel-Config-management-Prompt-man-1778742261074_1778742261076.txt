# أنتم تحتاجون نوعين من الـ backups وليس نوعًا واحدًا فقط

---

# 1) Full SQL Backup (الذي أضفتموه الآن)

هذا ممتاز لـ:

* disaster recovery
* migration
* restoring server state
* reproducibility

ويبقى:

```bash id="9b2ez8"
psql $DATABASE_URL < backup.sql
```

---

# 2) Research Export (منفصل)

وهذا مهم جدًا للبحث العلمي.

---

* users

* questions
* model outputs
* scores
* judges
* labels
* annotations

* CSV

# هذا الفصل مهم جدًا

## خلط operational backup مع research datasets

---

# نقطة ثانية مهمة جدًا

إذا عندكم:

* prompts
* evaluations
* judge reasoning

فأنصحكم تضيفون:

# Backup metadata manifest

مثل:

```json id="lcow5t"
{
  "platform_version": "1.4.0",
  "backup_created_at": "2026-05-14T03:57:39Z",
  "postgres_version": "16",
  "schema_version": "28",
  "included_tables": [...],
  "excluded_tables": [...]
}
```

---

# لماذا هذا مهم؟

لأن بعد سنة ستواجهون:

* schema drift
* migrations
* changed prompts
* changed judges

والـ manifest ينقذكم جدًا.

---

# أيضًا نقطة أمنية مهمة جدًا
